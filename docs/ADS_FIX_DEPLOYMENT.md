# Ads Not Showing on Production - Fix Documentation

## Problem Summary

Ads were showing on the development server but not on the deployed production site (wakeupnpc.com).

### Symptoms

- **Dev server**: Console showed "Square ads loaded: 2" and "Horizontal ads loaded: 2" with ad count of 104 total
- **Production**: Console only showed "✅ All content loaded in background" with no ad information
- **Result**: Zero ads displayed on production site

## Root Cause

The API endpoint `/api/content/ads` was using Node.js `fs` module to read markdown files directly from the `content/ads/` directory. This approach **does not work in Netlify's serverless environment** because:

1. Netlify Functions run in a serverless/Lambda environment
2. The `content/` directory with markdown files is not available at runtime
3. The file system operations were failing silently, returning empty data

## Solution

Changed the API endpoint to read from the pre-built `content-ads.json` file instead of parsing markdown files at runtime.

### Changes Made

1. **Updated `/server/api/content/ads.get.js`**:
   - Removed markdown parsing logic (gray-matter)
   - Now reads from `public/content-ads.json` (dev) or `.output/public/content-ads.json` (production)
   - Added environment detection for correct path resolution
   - Added debug logging to help troubleshoot in production

2. **Updated `/public/content-ads.json`**:
   - Ensured image paths match the actual files in `public/ads/`
   - Horizontal ads use: `768x90-ad-1b.png` and `768x90-ad-2b.png`
   - Square ads use: `378x378-ad-1.png` and `378x378-ad-2.png`

### How It Works Now

```
Development:
API reads from: /public/content-ads.json

Production (Netlify):
API reads from: /.output/public/content-ads.json
(This file is copied during build process)
```

## Files Modified

1. `/server/api/content/ads.get.js` - Complete rewrite to use JSON instead of markdown
2. `/public/content-ads.json` - Updated image paths to match actual files

## Deployment Steps

1. Commit the changes:

```bash
git add server/api/content/ads.get.js public/content-ads.json
git commit -m "Fix: Ads not loading in production - use JSON instead of markdown parsing"
git push origin main
```

2. Netlify will automatically deploy the changes

3. Wait for deployment to complete (~2-3 minutes)

4. Visit https://wakeupnpc.com and check browser console

5. Verify ads are displaying in the content wall

## Future Maintenance

If you need to update ads:

1. Edit the markdown files in `content/ads/`
2. Run `pnpm run regenerate-content` to update `public/content-ads.json`
3. Commit both the markdown files and the JSON file
4. Deploy

The build process automatically runs `regenerate-content` before building, so the JSON will always be up-to-date.

## Rollback Plan

If issues occur, you can rollback by:

```bash
git revert HEAD
git push origin main
```

This will restore the previous version (though it had the same issue).

## Additional Notes

- The `content-ads.json` file is generated by the `scripts/regenerate-content-json.js` script
- This script runs automatically during `pnpm run dev` and `pnpm run build`
- All ad images are stored in `public/ads/` and are included in the deployment
- The API endpoint now works identically in both dev and production environments
